## 🛠️ 工具使用规范

### 浏览器使用
**模式：** `profile="user"` (继承用户登录状态)

```yaml
# 正确用法
browser:
  action: open
  targetUrl: https://zhuanlan.zhihu.com/xxx
  profile: user  # ← 使用用户已登录的浏览器

# 后续操作继承同一浏览器
browser:
  action: snapshot
  targetId: [上次返回的 targetId]
  profile: user
```

**优势：**
- ✅ 继承知乎、Google 等网站的登录状态
- ✅ 无需重复登录
- ✅ 保持会话连续性

**注意：**
- 首次使用需要用户确认浏览器 attach
- 同一任务尽量复用 targetId，避免重复打开

### 文件操作
- 读取：`read` 工具
- 写入：`write` 工具（创建/覆盖）
- 编辑：`edit` 工具（精确替换）
- 目录：`exec` 工具（PowerShell 命令）

### 网页抓取
- 简单页面：`web_fetch` (快速提取文本/Markdown)
- 复杂页面：`browser` + `snapshot` + `act` (需要登录/交互)

## 📋 入库流程

```
1. 接收内容
   ↓
2. 判断来源 (网页/文档/用户输入)
   ↓
3. 抓取内容 (browser real 模式 / read)
   ↓
4. 分类整理 (技术知识/工作流程/业务信息/经验教训/外部导入)
   ↓
5. 格式化 (添加元数据：标题、来源、时间、标签)
   ↓
6. 存储 (写入 knowledge/对应分类/)
   ↓
7. 更新索引 (刷新 INDEX.md)
   ↓
8. 确认反馈 (向用户汇报入库结果)
```
