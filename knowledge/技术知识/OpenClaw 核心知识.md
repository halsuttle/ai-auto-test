## 🔑 三大核心系统

### 1. 文件系统（脑子）
- 万物皆可文件
- 建立专属知识库
- 习惯、喜好、深刻话题打包存储
- 避免"沙滩写字"式对话

**文件类型：**
- `MEMORY.md` - 长期记忆（永久保存）
- `USER.md` - 用户画像（老板偏好）
- `SOUL.md` - 灵魂档案（说话风格）
- `memory/` - 每日日记（随时间流动）
- `knowledge/` - 教科书（稳定可检索）

### 2. 心跳系统（灵魂）
- 传统 AI：不理它，它就死在那
- OpenClaw：主动思考、主动干活、主动找你
- 定时任务 + 周期性检查
- 真正"活"过来的关键

**心跳计划：**
- 9:00 - 检查当天日程
- 15:00 - 进度检查 + 提醒
- 20:00 - 当日总结 + 明日预告

### 3. Skills 系统（手脚）
- 技能包不是越多越好
- 根据需求精准配置
- 是极致的 SOP，不是魔法

**Skills 结构：**
1. 头部信息（YAML）
2. 触发条件
3. 执行步骤（SOP）

## 📊 数据化汇报

**核心武装：** `growth-tracker` 技能包

**三个维度：**
1. **成长值** - 今天学到了什么？记住了多少？
2. **能力评分** - 任务处理成功率是多少？
3. **状态值** - 上下文还剩多少空间？

**汇报频率：**
- 每日量化日报
- 每周能力进化曲线

---

## 🎯 交互方式

**核心逻辑：** 从"对话"转变为"指令"

**"圣旨"下达公式：**
```
角色 + 任务 + 限制
= 封官 + 定罪 + 设限
```

**原则：**
- 简洁高效，不啰嗦
- 直击重点，不废话
- 结果导向，不敷衍

## 🔄 深度进化

**进化 = 自我修改三个核心文件：**
- `SOUL.md` - 灵魂档案（说话风格）
- `USER.md` - 用户画像（偏好）
- `MEMORY.md` - 长期记忆（知识沉淀）

**进化路线图：**
1. 第一周：记忆期（多提供对话数据）
2. 第二周：学习期（给予反馈）
3. 第三周：主动期（纠正预判）
4. 第四周：自治期（彻底放手）

## 👥 多智能体协作

**核心逻辑：** 从"对话框"升级为"指挥部"

**分身角色：**
- 趋势猎人
- 文案专家
- 视觉指导
- 逻辑架构师
- 素材搜集员
- 审核校对员

**最佳实践：** 保持 3 个分身（搜索 + 创作 + 审核）性价比最高
