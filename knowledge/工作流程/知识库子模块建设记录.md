### 3. 浏览器使用规范

**位置：** `TOOLS.md`

| 模式 | 参数 | 用途 |
|-----|------|------|
| 隔离模式 | `profile: "openclaw"` | 默认，无登录状态 |
| **用户模式** | `profile: "user"` | **继承用户登录状态** ✅ |
| 扩展模式 | `profile: "chrome-relay"` | Chrome 扩展附加 |

**使用示例：**
```yaml
browser:
  action: open
  targetUrl: https://zhuanlan.zhihu.com/p/xxx
  profile: user  # ← 继承登录状态

# 后续操作复用同一浏览器
browser:
  action: snapshot
  targetId: [上次返回的 targetId]  # ← 保持会话
```

## 💡 关键决策

### 决策 1：为什么需要子 agent？
**原因：**
- 知识库管理是持续性工作，需要专职负责
- 子 agent 可以独立维护索引、检查元数据
- 主 agent 可以专注于其他任务，需要时调用小知

### 决策 2：为什么用 `profile: "user"`？
**原因：**
- 知乎、Google 等网站需要登录
- 继承用户登录状态避免重复登录
- 减少老板的交互成本

### 决策 3：为什么命名为"小知"？
**原因：**
- 简洁好记，符合中文命名习惯
- "知"代表知识，体现职责
- 拟人化命名，增强协作感
